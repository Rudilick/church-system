## 프로젝트 개요

교회 사무장/사역자/교역자 전용 교적·재정·소통 통합 관리 시스템.
교인이 직접 사용하지 않음 — 내부 관리자 전용 웹 애플리케이션.
인터넷이 되는 어디서나 접속 가능한 클라우드 배포 목표.

---

## 기술 스택

| 영역 | 기술 |
|------|------|
| Frontend | React 18 + Vite + React Router v6 |

## 전체 기능 명세

### 1. 교적 관리
- 교인 등록 / 수정 / 삭제
- 증명사진 업로드 및 전반적 활용
- **가족관계 시각화**: 교인 클릭 시 동그란 프로필 타일 + 가계도 맵핑
  - 타일에 이름, 성별, 나이 표시
- **공동체 소속 시각화**: 소속 공동체 클릭 시 구성원 얼굴 타일 구조화 표시
  - 예: 3여전도회, 이음공동체, 2지구1지역 등
- 분가 / 합가 처리
- 동명이인 구분
- **주소/직장/학교 + 지도 연계**: 교인 상세 페이지에서 다음지도 or 네이버지도 표시
- 생일 알림 (사역자에게 자동 알림)

### 2. 공동체 / 구역 관리
- 구역 / 지구 / 공동체 편성
- 구역장 계정 (추후 앱 연동 가능성 있음)
- 구역원 출석 및 기도제목 관리

### 3. 심방 관리
- 심방 기록 입력 (날짜, 대상 교인, 내용, 담당 교역자)
- 담임목사님 열람 권한 (권한 분리)

### 4. 출결 관리
- 예배별 출석 입력 (주일 1부/2부/3부, 수요, 새벽 등)
- **QR 출석 체크**: 브라우저 카메라로 QR 스캔 (별도 앱 설치 없음)
  - 교인은 종이/카드 QR 소지
  - 관리자 웹에서 카메라 켜서 스캔
- 출결 통계 (주간/월간/연간)
- 연령별 / 부서별 출석 통계 보고서

### 5. 헌금 관리
- **헌금 입력 UX 최적화**:
  - 이름 자동완성 (탭/방향키/엔터로 선택)
  - 입력할 때마다 입력 수 카운터 표시 (봉투 수 대조용)
- 헌금 종류: 십일조, 감사, 건축, 선교 등 항목별
- 헌금 이력 조회 (교인별 / 기간별)
- **기부금 영수증 자동 생성** (PDF)

### 6. 예산 / 장부 관리
- 수입 / 지출 거래 입력
- **영수증 파일 첨부** (사진 or PDF)
- 예산 항목별 관리
- **결산 보고서 자동 생성**
- 부서별 예산 현황
- 회계연도별 관리

### 7. 소통 / 행사

#### 스마트 캘린더
- 부서별 일정 등록 — 전 직원 공람
- **반복 일정**: 매주 / 격주 / 매월 N째주 설정
  - 예: "매월 3째주 남선교회·여전도회 모임"
- 필터링: 특정 부서만 보기 / 전체 보기
- 일정 충돌 감지 (같은 시간 같은 장소)
- 전 직원이 "어떤 부서가 언제 무슨 일정인지" 한눈에 파악

#### 내부 메신저
- 교역자 / 직원 간 1:1 및 그룹 채팅
- **스마트 링크 텍스트**:
  - `@이름` 입력 → 교인 프로필 페이지로 이동
  - `#부서명` 입력 → 해당 부서 페이지로 이동
  - 메시지 내 링크된 텍스트를 클릭하면 해당 데이터 페이지로 직접 이동
- 파일/사진 첨부
- 읽음 확인

#### 부서 페이지
- 부서별 전용 페이지 (교육부, 재정부, 친교부, 전도부, 청소년부 등)
- 부서 구성원 목록
- 부서 일정 (캘린더 연동)
- 부서 공지 / 게시판
- 부서 예산 현황 (재정 파트 연동)

#### 단체 문자 발송
- 대상 선택 (전체 / 구역별 / 부서별 / 개별)
- 외부 SMS API 연동 (알리고 or 솔라피 — 확정 후 연동)
- 발송 이력 관리

#### 스마트 요람
- 웹/앱에서 보는 디지털 요람
- 교인 사진 + 기본 정보 카드 형태

## DB 테이블 큰 그림

### 교적 파트
- `members` — 교인 기본정보, 사진 경로, 생일, 주소, 직장, 학교
- `families` — 가족 관계 매핑 (member_id, related_member_id, relation_type)
- `communities` — 공동체/구역/지구 정보
- `member_communities` — 교인↔공동체 다대다

### 예배/출결 파트
- `services` — 예배 일정 (주일1부, 2부, 수요 등)
- `attendances` — 출석 기록 (member_id, service_id, date, method)
- `qr_tokens` — 교인별 QR 코드 토큰

### 재정 파트
- `offering_types` — 헌금 종류
- `offerings` — 헌금 입력 기록 (member_id, type_id, amount, date)
- `budget_categories` — 예산 항목 (부서별)
- `transactions` — 지출/수입 장부 (amount, category_id, date, memo)
- `receipts` — 영수증 파일 첨부 (transaction_id, file_path)
- `fiscal_years` — 회계연도별 결산

### 소통 파트
- `pastoral_visits` — 심방 기록 (member_id, pastor_id, date, content)
- `events` — 캘린더 행사 (title, department_id, start, end, recurrence_rule)
- `departments` — 부서 정보
- `messages` — 내부 메신저 메시지
- `message_rooms` — 채팅방
- `sms_logs` — 문자 발송 이력

### 인증 파트
- `users` — 시스템 사용자 (사역자/직원 계정)
- `roles` — 권한 (admin, pastor, staff, section_leader 등)

## 개발 우선순위 (1단계부터)

1. **DB 스키마 작성** (PostgreSQL migration 파일)
2. **교적 관리** — 교인 CRUD + 사진 업로드 + 가족관계 시각화
3. **공동체 관리** — 구역/지구 편성 + 구성원 타일 시각화
4. **출결 관리** — 예배별 출석 입력 + QR 스캔 + 통계
5. **헌금 관리** — UX 최적화 입력 + 이력 조회 + 기부금 영수증
6. **예산/장부** — 거래 입력 + 영수증 첨부 + 결산 보고서
7. **소통** — 캘린더 + 메신저 + 부서 페이지 + 단체문자

## 참고 사항

- 기존 디모데 시스템을 기반으로 하되, UX를 대폭 개선하는 것이 목표
- 교인이 직접 사용하는 기능은 없음 (내부 관리자 전용)
- 구역장 앱은 추후 React Native로 연동 예정 (지금은 웹만)
- 지도 연동은 카카오맵 API 또는 네이버지도 API 사용 예정
- SMS API는 현재 사용 중인 서비스 확인 후 연동 (알리고 or 솔라피)
